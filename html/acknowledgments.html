Actimetre System

Actimetre is short for "Activity Metre". It is intended to measure the activity of lab animals, typically rats in cages, using commodity electronic components. For a hardware cost of less than $10 per cage, the system can record 6 axes of activity (acceleration XYZ and gyroscope XYZ) at up to 100Hz. A centralized dashboard helps monitor and manage the system, which can theoretically contain up to 35000 sensors. The entire system is fairly robust to disruption, except the Central component which is assumed to be highly available. Installation and configuration requires some amount of technical knowledge.

Acknowledgments

Actimetre System makes use of the following hardware and software components.

Actimetre

Hardware

  • S2 mini and S3 mini boards by LOLIN
  • Based on the ESP32-S2 and ESP32-S3 chips manufactured by Espressif Systems
  • Sensor breakout board GY-521 based on MPU-6050 by Invensense, a TDK company

Software

  • Arduino IDE with Espressif's ESP32 Arduino Core

Actiserver

Hardware

  • amd64 based Ubuntu 22.04 system optionally with RTL88x2AU/BU WiFi chip on USB, with drivers from morrownr
    or
  • ARM based Debian Bullseye or Ubuntu 22.04 system with on-board WiFi support.
    Tested on Raspberry Pi 3, Pi 4, Compute Module 4; NanoPi Neo3 with USB WiFi dongle; OrangePi Zero2.

Software

  • OpenJDK-17 running Kotlin code written on IntelliJ IDEA
  • On Raspberry hardware, the Embedded Linux JVM Debugger plug-in by asebak was used

Acticentral

Acticentral is really independent of hardware. It was tested on various single-board computers, Linux PCs, and AWS Lightsail.

Software

  • Apache2 for HTTP
  • Python3 for the main script
  • Matplotlib for graphs
  • Yattag for HTML generation with Python

Generic software

  • GNU Emacs for code editing
  • KiTTY and PuTTY as SSH and UART clients

History

Actimetre System was created by Boris Lamotte D'Incamps and Jay Han in the Spring of 2023, when Jay left his high-paying job for some time off and Boris was stuck debugging his I2C driver on a Raspberry Pi. Jay's idea of a time off was to write low-level C code, and Boris' way to get unstuck was to hire Jay as an intern (read: feed him). The system grew during the following few months to become the distributed system in place today.
